> 작성일: 2026-05-19
> 상태: **검토 중 (미반영)** — 영업 정책 결정 후 진행 여부 판단
> 관련 작업: 페이지 5 "05. 금융 모델별 수익성 비교 분석" 표의 수익배분형 컬럼

## 배경

수익배분형 컬럼을 페이지 5 표에 추가하면서 "1 REC (1,000 kW)" 행과 "REC수익/연간" 행을 **빈 셀(EmptyCell)** 로 두었음. 이유와 향후 처리 방향 정리.

---

## REC 도메인 기초 (간단 정리)

### REC란?
- **REC = Renewable Energy Certificate (신재생에너지 공급인증서)**
- 한국 **RPS 제도** 하에서 발급
- 대형 발전사(한전 산하 발전 자회사 등 18개 의무자)는 매년 발전량의 일정 비율(2024년 13%, 점진 증가)을 신재생에너지로 채워야 함
- 직접 신재생 발전소 짓거나, 다른 신재생 발전사업자의 REC를 매입해서 의무 충당

### REC 단위
- **1 REC = 1 MWh (= 1,000 kWh)** 의 신재생 전력 인증

### 발전 사업자의 REC 수익 구조 (이중 매출)
1. **SMP**: 전기 그 자체를 한전 등에 판매 (도매전력가격)
2. **REC**: 발전량만큼 발급된 인증서를 RPS 의무자에게 별도 판매

→ 영업 자료에서 "한전 판매 단가 = SMP + REC" 통합 단가로 표기되는 이유

### REC 가중치
- 일반 지상 태양광 (1MW 이하): 1.2
- 일반 지상 태양광 (1~3MW): 1.0
- 건물 옥상 (자가용/임대): **1.5**
- 수상 / 영농형: 1.5
- 임야: 0.5
- **ESS 연계 태양광: 4.0~5.0** (코드의 `REC 5.0`이 이걸 가리킬 가능성)

### REC 평균가격
- 코드 기본값 `recAveragePrice = 80` (천원/REC) = **약 8만원/REC**
- 한국 REC 시장 분기별 변동 (대략 5만~10만원/REC)
- ConfigModal에서 사용자가 변경 가능

## 현재 코드에서 수익배분형 REC 처리 상태

### 베이스 매출 (이미 SMP+REC 통합 단가 사용 중)
```ts
shareSalesRevenue = initialAnnualGen × config.unit_price_kepco
                  = 연간 발전량 × 210원/kWh
```

- `unit_price_kepco` = 210원/kWh
- ConfigModal 라벨: **"한전 판매 단가 (SMP+REC)"** → SMP + REC가 합쳐진 통합 단가
- 즉 **REC 가치가 베이스 매출에 이미 녹아들어가 있음**

### 표의 REC 관련 행은 빈 셀
| 행 | 다른 컬럼 (자기자본/RPS/팩토링/RE100/구독) | 수익배분형 |
|---|---|---|
| 1 REC (1,000 kW) | 환산 REC 개수 표기 | ❌ 빈 셀 |
| REC수익/연간 | REC × 단가 = 부가 수익 | ❌ 빈 셀 |

---

## 빈 셀로 둔 두 가지 이유

### 이유 1. 다른 컬럼들의 REC 행 의미가 일관되지 않음

코드 검증 결과 컬럼마다 산식이 다름:
```ts
// 자기자본/RPS/팩토링: 영업이익 ÷ REC 단가 = REC 환산값
rec_1000_common = annualOperatingProfit / recPrice / 1000;

// RE100연계: 한전 판매분 발전량(MWh) → 실제 발급 REC 추정
rec_1000_rent = (capacityKw * 0.2 * solarRadiation * 365) / 1000;

// 구독 서비스: 구독 수익 ÷ REC 단가 = 또 다른 환산
rec_1000_sub = sub_revenue_yr / recPrice / 1000;
```

→ "1 REC" 행이 컬럼마다 다른 의미. 수익배분형에 일관된 산식 적용하기가 모호.

### 이유 2. 수익배분형에서 REC 발급권을 누가 갖는지 영업 정책 미정
- **1~15년차**: 회사가 발전사업자라 REC 발급권 보유 → 회사 부가 수익 (지금 표는 고객 화면이라 표기 X)
- **15년차 소유권 이전 시점**: 발전 설비와 함께 REC 발급권도 이전?
- **16~20년차**: 고객이 발전사업자가 됨 → 고객 REC

→ 영업 정책 확정 안 됨

---

## 옵션 정리

### A. 현 상태 유지 (REC 빈 셀)
**처리**: 그대로 둠

**장점**:
- 표 단순, 영업 자료 깔끔
- 베이스 매출(210원/kWh)에 REC가 이미 포함됐다고 설명 가능
- 추가 작업 없음

**단점**:
- "수익배분형이 REC를 못 받는다"는 오해 가능성
- 영업 메리트 표현 부족

---

### B. 회사 측 + 고객 측 REC 분리 표기 (수익배분형만 손봄)
**처리**: 수익배분형 컬럼의 REC 행에:
```
1~15년차: 회사가 REC 발급권 → 회사 부가 수익
16~20년차: 고객이 REC 발급권 → 고객 부가 수익
```

**장점**:
- 변경 범위 작음 (수익배분형 컬럼만)
- 16~20년차 고객 REC가 영업 메리트 추가 (이미 매출 100% + REC까지)

**예시 수치 (469 kW, 1.5 가중치, 8만원/REC)**:
- 연간 발전량 약 651 MWh × 1.5 = **977 REC/년**
- 1~15년차 REC 수익 (회사): 977 × 8만원 ≈ **연 7,800만원 (회사 부가 수익)**
- 16~20년차 REC 수익 (고객): 977 × 8만원 ≈ **연 7,800만원 (고객 부가 수익)**

**영업 메시지 강화**:
> "1~15년차에는 발전 매출의 50% + 회사가 REC를 별도 수익으로 가져가서 사업 운영을 안정화합니다. **16~20년차에는 매출 100% + REC까지** 본인 수익이 됩니다."

**단점**:
- 다른 컬럼들의 REC 행 의미와 더 일관성 깨짐 (이미 컬럼마다 다른데 또 다른 의미 추가)

---

### C. 전체 REC 행 일관성 재설계 (대규모)
**처리**: 모든 컬럼의 REC 행 산식을 통일
- 1 REC = 실제 발급 REC 개수 (발전량 × 가중치 / 1000)
- REC수익/연간 = 실제 REC × REC 단가
- 모든 컬럼이 같은 의미로 통일

**장점**:
- 가장 깔끔, 컬럼 간 비교 명확
- 표 신뢰도 높아짐

**단점**:
- 변경 범위 큼 (5개 컬럼 모두 산식 수정)
- 기존 자료와 수치 차이 발생 가능 → 영업적으로 혼란
- 검토·테스트 필요

---

## 추천 사항

### 가성비: B안 우선 검토
- 변경 범위 작고 영업 메리트 명확하게 추가됨
- 다른 컬럼 영향 없음
- "16~20년차에 매출 100% + REC까지"라는 강력한 영업 메시지

### 장기적: C안도 같이 고려
- 영업 자료 전체 신뢰도 측면에서는 C안이 가장 합리적
- 다만 우선순위는 낮음 (현재도 영업적으로 동작)

### 보수적: A안 그대로
- 지금 표 그대로 유지
- 베이스 단가에 REC 포함됐다는 설명만 추가
- 추가 작업 없음

## 향후 진행 시 체크포인트

1. **REC 가중치 결정**:
   - 옥상 임대형(1.5)? ESS 연계형(4.0~5.0)?
   - 수익배분형의 일반적 가중치는 1.5 (옥상 임대)일 가능성

2. **REC 발급권 이전 시점**:
   - 15년 설비 이전과 함께 REC 발급권도 자동 이전?
   - 아니면 별도 협의?

3. **REC 매각 책임**:
   - 16~20년차에 고객이 REC를 누구에게 판매?
   - 회사가 대행 판매 옵션?
